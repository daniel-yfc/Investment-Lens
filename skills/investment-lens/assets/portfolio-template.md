# 📊 投資組合輸入格式引導

## 🚦 格式選擇指引

| 你的情況 | 建議格式 |
|---------|---------|
| 已有系統匯出的 CSV，欄位含 CFI 代碼、成本、市值、配息 | **格式 A：CSV 匯入版** |
| 多平台持倉（國泰 + eToro + 基富通等），需稅務優化 | **格式 B：完整手填版** |
| 快速了解配置比例，不需要稅務細節 | **格式 C：快速版** |
| 投資新手、資料不完整、不確定從何開始 | **格式 D：對話式** |

> 💡 **報價更新**：組合輸入後，隨時可說「更新報價」或 `/update-quote`，系統將自動刷新可查詢標的的最新市值，並記錄本次更新的 `value_date`。

## 格式 A：CSV 匯入版（最快速）

*適合：已有持倉試算表，直接貼上 CSV 內容讓系統解析*

**CSV 標頭格式（固定欄序）：**

```
# value_date: YYYY-MM-DD HH:MM (UTC+8)
# last_updated_by: [手動填寫 / investment-lens/update-quote]
標的代碼, Google Finance, 標的名稱, 類別, CFI 代碼, ISIN, 計價幣別, 匯率,
單位數, 平均成本(原幣), 持有部位(原幣), 持有部位(台幣), 配息,
最新報價, 未實現報酬率, 交易成本(原幣), 未實現損益(原幣), 未實現損益(TWD),
資產類別, 分類細項, 交易平台, 備註
```

**`value_date` 填寫規則：**

| 情況 | 填寫方式 |
|------|---------|
| 手動查詢後填寫 | `2026-03-21 15:30 (UTC+8)` |
| 由 `/update-quote` 自動更新 | 系統自動填入，附加 `[auto]` |
| 境內基金（T+1 淨值） | `2026-03-20 (T+1 NAV)` |
| eToro 智慧組合（有報價） | `2026-03-21 (eToro platform)` |

**CFI 代碼欄位說明：**

本 CSV 採用 ISO 10962:2021 CFI 代碼標準標記每個持倉的資產類型。完整對照表見 `references/asset-classification.md`。常見代碼速查：

| CFI 代碼 | 資產類型 |
|---------|---------|
| `ESVUFR` / `ESVTFR` | 已開發市場大盤股（台股、美股、歐股） |
| `EDSXFR` | 已開發市場個股（中概股 ADR，如 BABA、GOOG） |
| `ESNXFR` | 已開發市場個股（RTH 盤後交易標的） |
| `CEOGDU` | 中小型股 / 主題型 ETF（CIBR、SKYY、VBK 等） |
| `CEORDU` | 大盤指數型 ETF（VOO、VEA、IEFA 等） |
| `CEOBDU` | 投資等級公司債 ETF（00720B） |
| `CMOBDU` | 境內債券基金（元大10年投資級企業債） |
| `CMOGEU` | 境內多重資產基金（元大加權指數型、復華傳家等） |
| `CFOGEU` | 目標日期基金（國泰泰享退 2049） |
| `CIOGLU` | 對沖基金類策略（Momentum-LS） |
| `CIOGMU` | 多重資產智慧組合（The-Chameleon、Target2030-FT） |
| `CIOIMU` | 資本保護型產品（Target2030-FT Capital Guarantee 版）|
| `CIOGBU` | 數位資產（BTC） |

**系統處理流程：** 直接將 CSV 全文貼至對話框，系統將自動：
1. 讀取 `value_date` 判斷報價新鮮度（超過 3 個交易日自動提醒更新）
2. 依 CFI 代碼映射至全天候五大資產類別
3. 換算各持倉台幣市值並計算配置比例
4. 識別台股配息持倉（`配息` 欄 > 0）並啟動二代健保試算
5. 識別海外持倉（eToro + 中信智投）並加總海外所得
6. 識別 eToro 智慧組合並進行內部持倉解構

**報價過期警示：**

| 距 `value_date` 天數 | 警示 | 系統行為 |
|---------------------|------|---------|
| ≤ 1 個交易日 | ✅ 新鮮 | 直接分析 |
| 2–3 個交易日 | 🟡 稍舊 | 提示建議更新，可選擇繼續 |
| 4–7 個交易日 | 🟠 過期 | 警告，建議先執行 `/update-quote` |
| > 7 個交易日 | 🔴 嚴重過期 | 強制提示，分析結論加註「基於過期報價，僅供參考」 |

## 🔍 關鍵資訊說明

### 為什麼需要「CFI 代碼」？

ISO 10962:2021 CFI 代碼是國際標準的金融工具分類系統，Investment Lens 使用它來：
- 自動映射持倉至全天候五大資產類別（不依賴「資產類別」文字欄位，避免標記不一致）
- 識別台灣稅務處理方式（境內 vs 境外）
- 識別配息性質（現金配息 vs 累積型）
- 識別特殊結構型產品（如 `CIOIMU` = 資本保護型）

完整 CFI 對照表見 `references/asset-classification.md`。

### 為什麼需要「配息」欄位？

二代健保補充保費以**單筆實際收到的配息金額**為計算基礎：

$$\text{應繳補充保費} = (\text{單筆配息} - NT\$20{,}000) \times 2.11\%$$

### 為什麼需要「年收入稅率」？

$$\text{【稅率 20\%】} \quad 100{,}000 \times 20\% - 100{,}000 \times 8.5\% = 11{,}500 \text{ 元（稅後 88.5\%）}$$

低稅率 → 台股高股息 ETF 優先；高稅率 → 海外累積型 ETF 更有利。

## ❓ 常見問題

**Q：CFI 代碼填錯會影響分析嗎？**
A：會。系統優先以 CFI 代碼判斷資產類別，錯誤代碼可能導致全天候映射偏差。若不確定，可在備註欄標記「CFI 待確認」，系統將改用「資產類別」文字欄位作為備援。

**Q：BTC 顯示 -99.97% 報酬率是怎麼回事？**
A：持倉數量極小（約 0.0046 BTC），成本為 $118,990.80 但市值僅 $4,643，推測為早期高價買入後大幅縮減的殘值部位。建議確認是否已打算平倉，若帳面虧損可用於抵扣海外所得。

**Q：eToro 智慧組合現在有實際報價了，還需要手動確認嗎？**
A：需要確認更新頻率。eToro 智慧組合報價更新頻率低於個股，建議每月登入確認一次，特別是市場大幅波動後。

**Q：組合裡的加權指數型基金和 0050 / ETF 連結基金是否有重複計算？**
A：是的。元大台灣加權指數型基金（基富通 + 元大投信）與元大台灣50 ETF 連結基金，均追蹤台灣加權指數，與 0050 高度重疊。在計算全天候配置比例時，這三者應合併計算為「台股大盤」部位，避免重複計算導致股票比重虛高。

**Q：不記得確切持股數量怎麼辦？**
A：提供大概市值即可，系統用最新報價反推持股數量進行試算。

**Q：持倉超過 40 個標的，需要全部列出嗎？**
A：只需列出單一標的佔總資產 > 2% 的主要持倉；零碎部位（< 2%）可合併為「其他小額持倉約 NT$X 萬」。
