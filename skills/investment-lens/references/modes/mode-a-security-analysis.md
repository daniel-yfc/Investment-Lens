# Mode A — Security Analysis

Load this file when the task involves a single stock, ETF, index, or crypto asset,
or when performing qualitative valuation and thesis review.

---

## When to Use Mode A

- Single stocks (any market cap, any geography)
- ETFs and index funds
- Crypto assets
- Qualitative valuation framing and thesis review
- Earnings analysis and company deep-dives

## References to Load (as needed)

Load only what is relevant to the specific request. Do not load all files at once.

| Reference | When to load |
|-----------|-------------|
| `references/asset-classification.md` | Always — needed to frame the asset type |
| `references/market-asset-analysis.md` | For market/macro context framing |
| `references/tech-earnings-deepdive.md` | For technology sector earnings analysis |
| `references/investing-philosophies.md` | For thesis framing (value, growth, GARP, etc.) |
| `references/investing-philosophies-ref.md` | Deep reference for philosophy frameworks |
| `references/valuation-models.md` | When applying valuation multiples or models |
| `references/macro-liquidity.md` | When macro or liquidity context is relevant |
| `references/bias-checklist.md` | Always run before finalizing any security view |
| `references/market-bias-checklist.md` | When market-wide framing is involved |

## Core Output Requirements

1. Lead with recommendation or core judgment.
2. Separate thesis, counter-thesis, and key risks.
3. Use scenario framing for valuation-sensitive outputs.
4. State confidence level and valid-as-of date.
5. Run bias checklist before finalizing.

## Quant Handoff Trigger

Escalate to `quant-analysis` skill if the request explicitly requires quantitative modeling.
See `references/quant-handoff.md` for escalation protocol.
